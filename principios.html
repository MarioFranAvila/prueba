Los 10 principios de usabilidad de Jakob Nielsen


  1. Visibilidad del estado del sistema

    2. El sistema o página web, debe de proporcionar información al usuario acerca de lo que esta ocurriendo en determinados procesos. De esta manera, el usuario tendrá acceso a una retroalimentación continua acerca de los resultados de un proceso específico.

    Ejemplo: Al adjuntar un archivo en un correo utilizando Gmail, podemos conocer el progeso al adjuntar el archivo



  2. Relación entre el sistema y el mundo real

    3. El sistema o sitio web debe de utilizar un lenguaje natural que cualquier usuario pueda comprender, es decir, en lugar de utilizar palabras, frases o conceptos técnicos, lo que se sugiere es el uso de expresiones que resulten familiares y comprensibles para el usuario.

    Ejemplo: El usuario comete un error en la búsqueda de un repositorio, la página no existe, el navegador mostrará el error 404, el usuario puede no conocer este termino. Github muestra un mensaje personalizado de forma que indica al usuario que la página no se encontró.



  3. Control y libertad del usuario

    4. Este principio se enfoca en que el sitio web brinde al usuario la oportunidad de tener un control sobre su experiencia al navegar, de forma que tenga acceso a deshacer o revertir acciones o procesos que se ejecutaron por error.

    Ejemplo: Un usuario de la página "I Love PDF" adjuntó el archivo equivocado a convertir, podrá eliminarlo y seleccionar el correcto.



  4. Consistencia y estándares

    5. Parte de la satisfacción del visitante de la página, se puede regir por el uso de una interfaz que sea coherente y predecible, de forma que el usuario pueda interactuar de una forma intuitiva con el sitio. Los elementos de la página, deben de diseñarse de forma que sean consistentes, es decir, establecer y mantener convenciones lógicas.

    Ejemplo: Poder identificar de forma sencilla en una página web el apartado de búsquedas o la opción para cambiar de país, puede hacerse con el uso de un diseño que permita al usuario intuir el como realizar estas acciones. Por otro lado, el uso del icono de menú hamburguesa le brinda al usuario la posibilidad de identificar rápidamente su propósito. Un ejemplo de un sitio web que cumple con esto es Amazon.



  5. Prevención de errores

    6. Este principio tiene como objetivo que el usuario no caiga en un error. El enfoque debe ser diseñar un sitio web de manera que se minimice al máximo la posibilidad de que el usuario cometa algún error. Si se previene y anticipa el error, el usuario puede ahorrarse la corrección del mismo, teniendo una mejor experiencia dentro del sitio.

    Ejemplo: Los navegadores como Google ofrecen al usuario la posibilidad de contar con asistencia al momento utilizar la barra de búsqueda, de forma que se pueda agilizar el proceso de búsqueda, mediante técnicas como autocompletar o alternar posibles resultados de búsqueda.



  6. Reconocimiento antes que recuerdo

    7. El objetivo de este principio es que el sitio web cuente con un diseño que brinde al usuario la posibilidad de reconocer e identificar las diferentes opciones o funciones dentro del sitio, de forma que el usuario no necesite recordar información entre diferentes secciones del sitio.

    Ejemplo: El uso de una barra o menú de opciones que cuente con iconos y texto que permitan al usuario reconocer fácilmente la función de cada apartado. Un ejemplo sería en el sitio web de Volaris, donde se cuenta con un menú intuitivo que el usuario no deberá recordar.



  7. Flexibilidad y eficiencia de uso

    8. Este principio tiene como fin que el diseño del sitio web permita un uso efectivo del mismo a un grupo amplio de usuarios, desde usuarios sin mucha experiencia hasta usuarios catalogados como expertos. Este principio contempla el uso de atajos o funciones más sofisticadas para usuarios más avanzados, de forma que optimicen o agilicen ciertas tareas. A su vez, debe contar con la posibilidad de que los usuarios primerizos puedan realizar las mismas tareas, aunque de forma más pausada y guiada.

    Ejemplo: Google permite a sus usuarios el uso de comados avanzados para obtener mejores resultados de búsqueda, esto mediante el uso de palabras clave. Un ejemplo sería utilizar el comando filetype para encontrar contenido en un formato específico y que contenga cierta palabra clave.



  8. Estética y diseño minimalista

    9. En este principio, "menos es más", ya que se pretende evitar la sobrecarga de información que pueda confundir al usuario o desviar su atención de la información más relevante. Utilizar un diseño minimalista para nuestro sitio web, mejorará significativamente la apariencia, a la vez que facilitará la navegación del usuario y procurará mantener un enfoque en cumplir su objetivo dentro del sitio, de una forma eficaz.

    Ejemplo: Una presentación inicial del sitio web que va directo al grano, Netflix es un claro ejemplo.



  9. Ayudar a los usuarios a reconocer, diagnosticar y recuperarse de errores

    10. Este principio se encarga de entregar al usuario un mensaje de error lo suficientemente claro, como para que el usuario logre identificar el problema, comprender su origen y pensar en una posible solución para el problema. La implementación de este principio en nuestro sitio web, mejorará los niveles de satisfacción de los visitantes, ya que ante un posible error, el grado de frustración del usuario será menor al lidiar y resolver errores.

    Ejemplo: El formulario de registro de Linkedin informa al usuario en caso de un error al ingresar la información.



  10. Ayuda y documentación

    11. Este principio pretende que el usuario pueda identificar fácilmente un apartado de ayuda en caso de requerirla o de que tenga algún problema al realizar alguna tarea. Esta sección debe de ser precisa y especifica, de manera que el usuario logre evacuar sus interrogantes y cuente con una guía con los pasos necesarios que brinde al visitante la oportunidad de comprender como utilizar adecuadamente el sitio.

    Ejemplo: El sitio web de CISCO, contiene un apartado de ayuda fácil de identificar, con espacios de preguntar frecuentes, una guía para inicio de sesión y un asistente virtual.
